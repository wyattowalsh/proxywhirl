✅ 5 FULLY SUPPORTED DEPLOYMENT PLATFORMS

1. Docker Compose (Local Development)
   • Multi-service stack with all dependencies
   • Health checks and auto-restart
   • Quick start: docker-compose -f deploy/docker-compose.yml up -d
   • Services: ProxyWhirl, Redis, PostgreSQL, Prometheus, Grafana

2. Kubernetes (Enterprise)
   • 13 production-grade manifests
   • Rolling updates, security hardening, RBAC
   • Quick start: kubectl apply -f deploy/kubernetes/
   • Features: HPA, NetworkPolicy, Ingress, persistent storage

3. Helm (Package Manager)
   • Production Helm chart with 100+ parameters
   • Environment-specific configuration
   • Quick start: helm install proxywhirl deploy/helm/
   • Supports: Values overrides, conditional resources

4. Terraform/AWS (Cloud Native)
   • Complete IaC for AWS infrastructure
   • ECS, RDS, ElastiCache, ALB, monitoring
   • Quick start: cd deploy/terraform && terraform apply
   • State management via S3 backend

5. systemd (Linux Server)
   • Native Linux service deployment
   • API and worker services with socket activation
   • Quick start: sudo systemctl start proxywhirl
   • Perfect for: Traditional server deployments

  Access: http://localhost:8000 (API), http://localhost:3000 (Grafana)

Option 2: Kubernetes (Recommended for Production)
  kubectl create namespace proxywhirl
  kubectl apply -f deploy/kubernetes/

Option 3: Helm (Package Manager)
  helm repo add proxywhirl .
  helm install proxywhirl deploy/helm/ \
    --namespace proxywhirl \
    --create-namespace \
    --values my-values.yaml

Option 4: Terraform/AWS (Cloud Native)
  cd deploy/terraform
  terraform init
  terraform plan -var-file=prod.tfvars
  terraform apply -var-file=prod.tfvars

Option 5: systemd (Linux Server)
  sudo cp deploy/systemd/* /etc/systemd/system/
  sudo systemctl daemon-reload
  sudo systemctl enable --now proxywhirl proxywhirl-worker
