# Resilience

Public proxy pools are volatile. ProxyWhirl layers four resilience mechanisms so applications keep making progress without manual babysitting.

## Components

| Layer | Role |
| ----- | ---- |
| **Cache** | Reduce repeated fetches and validation work |
| **Circuit breaker** | Stop routing through proxies that fail repeatedly |
| **Retry** | Back off and try alternate proxies on transient errors |
| **Rate limiting** | Cap request velocity to protect upstreams |
