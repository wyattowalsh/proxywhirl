# Concepts

ProxyWhirl has four recurring concepts:

| Concept            | Meaning                                                                                                     |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| Proxy lifecycle    | Fetch, parse, validate, enrich, rotate, retire.                                                             |
| Strategy selection | Pick a proxy according to load, latency, geography, cost, or session affinity.                              |
| Resilience         | Retries, circuit breakers, rate limiting, and cache layers absorb unreliable networks.                      |
| Security           | Credentials, target URL validation, private-network protections, and redaction controls reduce misuse risk. |

Generated strategy and source pages keep the concept docs tied to the current source code.
