ProxyWhirl exposes the same proxy rotation capabilities through language-native, HTTP, command-line, and agent-facing interfaces.

| Interface | Use when |
| --------- | -------- |
| [MCP Server](/docs/interfaces/mcp) | AI assistants need structured proxy tools |
| [REST API Deployment](/docs/interfaces/rest-api) | Services need HTTP access to pool management and rotation |
| [Clients](/docs/guides/clients) | Python applications can import `ProxyWhirl` or `AsyncProxyWhirl` directly |
| [CLI Reference](/docs/generated/cli-reference) | Operators need shell commands for fetch, source validation, and pool tasks |

Start with Python clients for embedded applications, REST for service boundaries, and MCP for assistant workflows.
